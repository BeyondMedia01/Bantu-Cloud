## Task 3: taxEngine — `grossUpNet()` solver

- [ ] **Step 3: Implement `grossUpNet` in `taxEngine.js`** (add before `module.exports`)

```js
/**
 * Gross-up solver: finds the gross salary G such that calculatePaye(G).netSalary ≈ targetNet.
 * Uses binary search — converges to within $0.01 in <40 iterations for any realistic salary.
 *
 * Accepts the same params as calculatePaye(), minus baseSalary (which is solved for).
 * Returns the same result object as calculatePaye(), with baseSalary set to the solved value.
 */
function grossUpNet({ targetNet, currency = 'USD', maxIterations = 50, tolerance = 0.01, ...payeParams }) {
  // Lower bound: net salary (no tax)
  // Upper bound: worst-case 40% + 3% AIDS levy → net ≈ gross × 0.57; use 0.50 for safety margin
  let lo = targetNet;
  let hi = targetNet / 0.50; // generous ceiling — covers high-earner edge cases

  let best = null;
  for (let i = 0; i < maxIterations; i++) {
    const mid = (lo + hi) / 2;
    const result = calculatePaye({ baseSalary: mid, currency, ...payeParams });
    const net = result.netSalary;
    if (Math.abs(net - targetNet) < tolerance) {
      best = result;
      break;
    }
    if (net < targetNet) lo = mid;
    else hi = mid;
    best = result;
  }
  return best;
}
```

- [ ] **Step 8: Invoke `grossUpNet` when `emp.grossingUp === true`**

Find the block where `fdsAvgPAYEBasis` is computed and the `calculatePaye` calls begin (around line 825). Just before the first `calculatePaye` call, add:

```js
// Gross-up: employer absorbs PAYE only — employee still pays NSSA, pension, medicalAid.
// The solver must find gross G where:  G - NSSA(G) - pension - medicalAid - PAYE(G) = baseSalary
// So: targetNet = baseSalary + effectivePension + medicalAid  (i.e. desired pre-PAYE-only net)
const effectiveBaseSalary = emp.grossingUp
  ? (() => {
      const isZIG = run.currency === 'ZiG';
      const cappedPension = pensionCap != null
        ? Math.min(pensionContribution, pensionCap)
        : pensionContribution;
      const medForGrossUp = isZIG ? 0 : (inputMedicalAidUSD || 0);
      const grossUpTargetNet = baseSalary + cappedPension + medForGrossUp;
      const solved = grossUpNet({
        targetNet: grossUpTargetNet,
        currency: isZIG ? 'ZiG' : 'USD',
        taxBrackets: isZIG ? taxBracketsZIG : taxBracketsUSD,
        annualBrackets: emp.taxMethod === 'FDS_FORECASTING' ? true : annualBracketsUSD,
        nssaCeiling: isZIG ? nssaCeilingZIG : nssaCeilingUSD,
        pensionContribution, pensionCap,
        medicalAid: medForGrossUp,
        taxCredits: elderlyCredit > 0 ? elderlyCredit : (emp.taxCredits || 0),
        nssaEmployeeRate, nssaEmployerRate,
      });
      return solved ? solved.grossSalary : baseSalary;
    })()
  : baseSalary;
```

Then replace `baseSalary` with `effectiveBaseSalary` in the subsequent `calculatePaye` calls for that employee.

**Why `targetNet = baseSalary + pension + medicalAid`:** `calculatePaye` defines `netSalary = cashEarnings − NSSA − pension − medicalAid − PAYE`. The employer absorbs only PAYE, so the employee should still net `baseSalary` after paying NSSA, pension, and medicalAid themselves. Setting `targetNet` to `baseSalary + pension + medicalAid` tells the solver to find the gross where the full `netSalary` (after all deductions) equals that sum — which means PAYE was the only extra absorbed by the employer.

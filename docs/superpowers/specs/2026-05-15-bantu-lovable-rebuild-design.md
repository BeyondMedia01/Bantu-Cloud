# Bantu Platform — Lovable Full-Stack Rebuild Design

**Date:** 2026-05-15
**Status:** Approved
**Scope:** Full rebuild of Bantu Payroll & HR SaaS using Lovable (React frontend) + Supabase (Postgres + Auth + Edge Functions) + Cloudflare (security layer)

---

## 1. Overview

Bantu is a Zimbabwe-focused Payroll & HR SaaS with native multi-currency support (USD/ZiG), deep statutory integrations (ZIMRA, NSSA, NEC), and a multi-tenant architecture serving payroll bureaus, SMEs, and enterprises.

The rebuild replaces the existing React/Node.js/Prisma stack with:
- **Frontend:** React (generated by Lovable)
- **Backend:** Supabase (Postgres + Row Level Security + Edge Functions for payroll calculations)
- **Security:** Cloudflare (WAF, rate limiting, DDoS protection, hiding Supabase URL)
- **Biometric integration:** Deferred to Phase 2

---

## 2. Architecture

```
Browser (React)
    │
    ▼
Cloudflare (WAF + Rate Limiting + DDoS)
    │
    ▼
Supabase
    ├── Auth (JWT, invitations, sessions)
    ├── Postgres (multi-tenant data + RLS)
    └── Edge Functions (payroll calculations, PDF generation, exports)
```

### Rate Limits (enforced at Cloudflare)
- Auth endpoints: 5 requests / 15 min
- API endpoints: 200 requests / 15 min

## 3. Multi-Tenancy Model

Three-level hierarchy:

```
Client (payroll bureau / org)
  └── Company
        ├── SubCompany (optional)
        ├── Branch
        └── Department
              └── Employee
```

All data is scoped to `company_id`. RLS policies enforce this at the database level — no application-level filtering required.

## 5. Row Level Security Strategy

Every company-scoped table has RLS policies:

```sql
-- Example: employees table
CREATE POLICY "users see own company employees"
ON employees FOR ALL
USING (
  company_id IN (
    SELECT company_id FROM users WHERE id = auth.uid()
  )
);
```

Access rules by role:
- **PLATFORM_ADMIN:** Uses service role key in Edge Functions only — never client-side
- **CLIENT_ADMIN:** Sees all companies under their `client_id`
- **COMPANY_USER:** Sees only their assigned `company_id`, further filtered by role permissions
- **EMPLOYEE:** Sees only rows where `employee_id = auth.uid()` (payslips, leave, attendance, documents)

## 7. Payroll Engine (Edge Functions)

All statutory calculations run server-side in Supabase Edge Functions. Core logic ported from existing `backend-v2` TypeScript:

- **PAYE:** ZIMRA tax bands with effective-date tax tables, 3 methods: `FDS_AVERAGE`, `FDS_FORECASTING`, `NON_FDS`
- **AIDS Levy:** 3% of PAYE
- **NSSA:** Employee + employer contributions with thresholds
- **NEC:** Industry council levies
- **SDF / WCIF / ZIMDEF:** Additional statutory contributions
- **Dual-currency:** Per-employee USD/ZiG split percentages applied to gross and net
- **Medical Aid Credit:** Informational display only, excluded from totals

Edge Functions for:
- `POST /functions/v1/calculate-payroll` — full run calculation
- `POST /functions/v1/generate-payslip-pdf` — HTML-rendered dual-currency payslip
- `POST /functions/v1/export-bank-eft` — CBZ / Stanbic / Fidelity formats
- `POST /functions/v1/export-zimra-p2` — ZIMRA monthly return
- `POST /functions/v1/export-nssa-p4a` — NSSA monthly return
- `POST /functions/v1/export-zimra-it7` — Annual IT7 (P16) certificates

**Delivers:** Full leave + attendance (no biometrics)

Covers: Leave policy config · Leave request workflow (submit → approve → reject) · Leave balance tracking · Monthly accrual (Supabase cron) · Shift configuration · Manual attendance entry · Public holiday management

**Delivers:** End-to-end payroll run with statutory deductions

Covers: Transaction codes · Payroll calendar · Run lifecycle (Preview → Submit → Approve → Process) · Dual-currency USD/ZiG · PAYE + AIDS Levy + NSSA + NEC + SDF + WCIF via Edge Functions · Loan repayment auto-deduction

### Prompt 5 — Payslips & Loans
**Delivers:** Payslips employees can view and download

### Prompt 6 — Compliance & Statutory Exports
**Delivers:** Full statutory compliance exports

Covers: ZIMRA IT7 / P2 · NSSA P4A · Tax table management · Bank EFT files (CBZ, Stanbic, Fidelity) · Motor vehicle benefit

## 10. Success Criteria

- All 4 user roles can log in and see their correct navigation
- A payroll bureau (CLIENT_ADMIN) can manage multiple companies from one login
- A full payroll run produces correct PAYE, NSSA, AIDS Levy for a dual-currency employee
- Payslips are downloadable as PDF with USD and ZiG figures
- ZIMRA P2 and NSSA P4A exports match the required format
- RLS prevents any cross-tenant data access
